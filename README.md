# scanports
Описание
--------
В данном проекте я практиковался с помощью **-Nmap-** в нахождении уязвимость и сканирование портов.
Проект воспроизводит CVE-2016-8869-70 в контролируемом lab‑окружении (Joomla X.Y).


Этика
---------
Все тесты выполнялись только на локальной машине/в Docker. Не проверяйте чужие сайты без разрешения.


---
# сканирование портов и сервисов
nmap -sV -p 80,443 -oN nmap_http.txt [target]

# http scan с NSE
nmap -sV --script=http‑vuln -p 80,443 -oN nmap_http_vuln.txt [target]





------


При поверхностом сканировании не было выявлено уязвимостей сайта



• **Базовый скан:**

nmap -sS -p- -T4 -oN all_ports.txt [target]
---

• **Поиск сервисов и версий:**

nmap -sV -p80,443 -oN http_services.txt [target]
---

•** NSE‑скрипты по http:**

nmap -sV --script=http-vuln*,http-enum -p80,443 -oN http_vulns.txt [target]

**Критическая уязвимость** - Joomla! version 3.6.4:
Joomla 3.6.4 была выпущена 11 августа 2016 года. С тех пор эта ветка CMS получила множество обновлений безопасности. Вся ветка Joomla 3.x достигла End of Life (EOL) 17 августа 2023 года и больше не получает обновлений безопасности.


**• Последствия:** Использование такой устаревшей версии Joomla почти гарантирует наличие множества известных, публично раскрытых и легко эксплуатируемых уязвимостей, таких как удаленное выполнение кода (RCE), SQL-инъекции, межсайтовый скриптинг (XSS), уязвимости в управлении файлами и правами доступа, а также проблемы в компонентах и расширениях. Злоумышленник, скорее всего, сможет найти и использовать эксплойты для этой версии, что может привести к полной компрометации веб-сайта и потенциально всего сервера.



**• Результаты http-enum:**
/logs/: Logs - Обнаружена директория с логами.

/administrator/manifests/files/joomla.xml: Joomla version 3.6.4 - Файл, явно указывающий версию Joomla.

/language/en-GB/en-GB.xml: Joomla version 3.6.4 - Еще один файл, подтверждающий версию Joomla.

/htaccess.txt: Joomla! - Обнаружен файл конфигурации .htaccess, характерный для Joomla.

/bin/, /cache/, /images/, /includes/, /libraries/, /modules/, /templates/, /tmp/: Все эти директории помечены как "Potentially interesting folder". Это стандартные директории Joomla, которые могут содержать важные файлы или быть точками входа для атак, если они неправильно защищены.
